Introduction API Endpoints Usage

Introduction

This documentation covers the setup and usage of the Node.js server for your application. The server provides a RESTful API for various functionalities.

API Endpoints

The following endpoints are available:

  • POST /check-email

    sends verification mail
    Data which would be sent (as json): 
    
                {
                    fname : //First name ,
                    lname : //Last name,
                    email : //email,
                    password : // password,
                    username : //username,
                }
    it response by:

       Status 401 (bad request): if email or username already exits
       Status 500 (Internal server error): if error occurs in the server
       Status 200 (Sucess): if email is sent.


  • POST /login

    sends login request
    Data which would be sent (as json): 
    
                {
                    password : // password,
                    username : //username but also it can be email
                    
                }
    it response by:

       Status 403 (Invalid login): for invalid logins
       Status 500 (Internal server error): if error occurs in the server
       Status 200 (Sucess) : for valid logins

    for valid logins it returns : 
    
                {
                    "username" : //username,
                }
